Projet universitaire · Web app 04 / 05

Plateformee-commerce Laravel

Rôle Développeur full-stack
Équipe 5 personnes
Durée 3 mois
Contexte BUT IAMSI
Laravel PHP HTML CSS JavaScript MySQL Agile / Scrum

Contexte

Projet universitaire réalisé en équipe de cinq sur trois mois, l'objectif était de concevoir une solution e-commerce complète couvrant les parcours principaux d'un utilisateur : navigation dans le catalogue, gestion des favoris, panier et passage de commande.

Le projet était organisé en sprints avec un backlog structuré, ce qui nous a permis de travailler de façon itérative et de livrer des fonctionnalités testables à chaque cycle.

Ce qu'on a construit

  • Structuration des besoins et découpage fonctionnel en sprints
  • Développement du catalogue produit avec filtres et navigation
  • Système de favoris persistants par utilisateur
  • Logique de panier - ajout, modification de quantité, suppression
  • Parcours commande de bout en bout
  • Interface web côté utilisateur
Point technique

La gestion du panier et des favoris

La partie la plus exigeante du projet. Le panier devait être cohérent entre les pages, persistant pour un utilisateur connecté, et réactif côté interface sans alourdir les échanges avec le serveur. Les favoris posaient une question similaire : synchroniser l'état entre la vue catalogue et la vue dédiée sans introduire de bugs de cohérence.

On a travaillé la session Laravel côté back et géré les interactions côté JS pour éviter les rechargements inutiles, un bon exercice d'articulation entre les deux couches.

Ce qu'on retient

Une approche projet structurée

Ce projet m'a appris autant sur le process que sur le code. Travailler à cinq sur un même codebase Laravel, avec des sprints définis et un backlog maintenu, a demandé une discipline de coordination qu'on n'apprend pas en faisant du code seul.

L'organisation en sprints a rendu le projet livrable : chaque itération avait un périmètre clair, ce qui évitait l'effet tunnel et permettait d'ajuster la priorité en cours de route.
