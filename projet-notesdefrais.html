Rôle Développeur
Durée 1 an
Produit ERP maison
Contexte ilem Group
Java Spring Boot Vaadin SQL ERP maison Azure DevOps

Contexte

Alternance d'un an chez ilem Group, une ESN qui développe et maintient son propre ERP en interne. La mission : concevoir et intégrer un module dédié à la gestion des notes de frais, directement dans une base de code existante, volumineuse, et déjà en production.

Le travail couvrait l'ensemble du cycle : compréhension des specs, développement du module, intégration dans l'existant, persistance des données et cycle de correction jusqu'à la stabilisation.

Ce qui a été développé

  • Lecture et compréhension des spécifications fonctionnelles
  • Développement des composants du module avec Vaadin et Spring Boot
  • Modélisation et intégration côté base de données
  • Connexion aux entités existantes de l'ERP
  • Correction de bugs et stabilisation en continu
Le vrai défi

Entrer dans un projet existant sans se perdre

La difficulté principale n'était pas technique au sens strict, c'était de monter en compétence sur une techno nouvelle (Java, Spring Boot, Vaadin) tout en s'intégrant dans un ERP maison développé sur plusieurs années, avec ses propres conventions, son architecture et ses dépendances implicites.

Au début, chaque action demandait de remonter des chaînes de code sans vraiment comprendre où ça menait. Le vrai tournant, c'est le moment où j'ai commencé à voir les relations entre les entités de l'application : comment les modules se parlaient, où vivait la logique métier, pourquoi certaines choses étaient faites ainsi. À partir de là, naviguer dans le projet est devenu naturel, et contribuer, efficace.

Ce qu'on retient

Lire un projet avant d'écrire du code

Cette alternance m'a donné un réflexe que je n'avais pas avant : passer du temps à comprendre avant de commencer à produire. Sur un projet existant, écrire du code sans avoir compris l'architecture crée plus de problèmes qu'il n'en résout.

Savoir lire un projet, ses relations, ses patterns, ses conventions, c'est une compétence à part entière, aussi importante que savoir écrire du code propre. Et c'est celle que j'ai le plus développée cette année.
